Grundeinstellungen / WYSIWYG Editor
- In den Grundeinstellungen gibt es neu die Option "Eigenen Style im WYSIWYG Editor verwenden".
Ist diese aktiviert, wird die Datei "style.css" des aktuellen Standard-Template in den WYSIWYG Editor
eingebunden. Somit kann der Style bereits schon während dem editieren geladen und angeschaut werden.
Wird diese Option deaktivert, wird der normale WYSIWYG Editor Style geladen.
- Aus sicherheits- und benutzerfreundlichen Gründen wird nun standardmässig jeder Text der in den
WYSIWYG Editor kopiert wird, als Plaintext eingefügt. (FCKConfig.ForcePasteAsPlainText = true;)

Allgemein
- Neue Platzhalter für Datumsfragmente:
  DATE_YEAR, DATE_MONTH, DATE_DAY, DATE_TIME
- Neu, über den Modul Manager installierte Module, erscheinen nicht in der
  Inhalts Navigation.
- Thumbnail-Erstellung von Transparenten Bildern wird nun unterstützt.
- Das Sprachselektionsmenu hat nun eine "selected" CSS-Klasse gesetzt für die aktive Sprache
- Das Handy-Template wird nun dem seitenspezifisch gesetzten Template vorgezogen.


File Browser
- Beim Datei-Upload kann zwischen dem normalen Upload oder der Verwendung des
  File Uploader Moduls (sofern vorhanden) ausgewählt werden.

Content Manager
- Der CSS-Name und das "target" für Navigationslinks können nun auch an Unterseiten vererbt werden
- Editeren einer Seite in mehreren Sprachen auf einmal
- Vorschau-Funktion
- Die Seiten Struktur wird nun für alle Sprachen synchronisiert
- Seiten können per drag&drop strukturiert werden
- Neuer WYSIWYG Editor (siehe CKEDITOR unten)
- Eine Contentseite kann nun auf eine andere Sprache verweisen und deren
  Content verwenden, damit nicht Seiten mit gleichem Inhalt in mehreren
  Sprachen erfasst werden müssen.

Benutzerverwaltung
- Beim Erfassen eines neuen Benutzerkontos kann automatisch eine persönliche
  Kategorie beim Digital Asset Management Modul erstellt werden.
- Das Aktualisierungsintervall der Benutzersitzungen in der Datenbank
  kann definiert werden.
- Bei der Auflistung der Benutzer wird der Link 'zum Adressbuch hinzufügen'
  nur dann angezeigt, sofern der jeweilige Benutzer nicht bereits im Adressbuch
  enthalten ist.
- Für jede Benutzergruppe kann eine Homepage definiert werden, auf welche sich
  die dazugehörigen Benutzer nach erfolgreicher Authentifizierung laden
  sollen.
- Für jeden Benutzer kann eine primäre Benutzergruppe definiert werden.
- Der Platzhalter [[ACCESS_USER_PRIMARY_GROUP]] gibt den Namen der primären
  Benutzergruppe aus.
- Die Registrierungsmaske kann optional um ein CAPTCHA erweitert werden.
- Die Registrierungsmaske kann optional um die Aufforderung der AGB's zu
  Akzeptieren erweitert werden.
- Mit dem neuen Platzhalter [[USER_COUNT]] kann die Anzahl Mitglieder
  in der index.html des Layouts ausgegeben werden.
- Profil Eigenschaften vom Typ Auswahlmenü können als Pflichtfeld definiert
  werden.
- Die Profil Eigenschaften Geschlecht, Anrede, Land und Webseite werden nur
  aufgelistet, sofern eine Option ausgewählt wurde.
- Für das Profilbild-Thumbnail kann ausgewählt werden, ob das Bild auf die
  Thumbnail-Grösse zugeschnitten oder skaliert werden soll.

Newsletter
- Neu kann der Newsletter vom Empfänger direkt im Browser aufgerufen und betrachtet
  werden. Hierzu muss der Platzhalter [[display_in_browser_url]] wahlweise in 
  das Newsletter-Template und/oder Content eingearbeitet werden.
  Der Platzhalter generiert beim Versand dann automatisch einen individuellen Link
  pro Empfänger.

Kontakte Manager
- Alle Benutzerprofil Eigenschaften des Besuchers (sofern dieser Authentifiziert
  ist) können anhand des Platzhalter Schemas
  [[ACCESS_PROFILE_ATTRIBUTE_ATTRIBUTE-ID]] als vordefinierte Werte bei den
  Formular Feldern verwendet werden. Zusätzlich stehen die folgenden
  Benutzerkonto Eigenschaften zur Verfügung:
    E-Mail: [[ACCESS_USER_EMAIL]]
    Benutzername: [[ACCESS_USER_USERNAME]]
    ID des Benutzers: [[ACCESS_USER_ID]]
- Hochgeladene Dateien werden bei einer Kontaktanfrage im Benachrichtigungs-
  E-Mail als Dateianhänge angefügt.
- Einige vorgenerierte <br />-Tags, welche sich für Webdesigner als eher störend erwiesen, wurden entfernt.
- Heraufgeladene Dateien werden nun auch unter UTF-8 korrekt umbenannt
- HTML-Code der generierten Formulare wurde bereinigt (fieldset nun innerhalb form)

Korrespondierende Mehrsprachigkeit
- Auch bei mehreren Sprachen haben alle Seiten die gleiche ID

- Neues Verhalten: Einstellung < 1: keine Headlines. Einstellung > 50: 50 Headlines.

  der Rubriken als Baum-Struktur (Explorer Art).
